### What this document is

A shared plan for building Round 2 of the manifesto website. It is written so
that at any point — in a steering meeting, mid-build, or six weeks from now —
anyone can see what is finished, what is in progress, and what is waiting on a
decision. That visibility is the point: it keeps expectations honest on both
sides.

### How to read it

Every milestone has an **owner**:

- **`[BLKOUT]`** — work BLKOUT builds or delivers.
- **`[BMHWA]`** — a decision or input only the steering committee can provide.
- **`[JOINT]`** — a sign-off or working session needing both.

**Content sign-off:** Ariel Breaux is BMHWA's single point of contact for
signing off new content sections. Content `[BMHWA]` and `[JOINT]` approvals
route through her rather than the full steering committee — this keeps the
gates moving.

Some milestones are **⛔ Gates**: the project cannot move past them until a
`[BMHWA]` decision is made. Gates are listed in their own summary table at the
end. If a gate is not cleared, the work behind it simply waits — that is
expected, and it is visible.

**On the dates:** three are firm anchors — plan sign-off **30 May**, Phase 1
live **30 June**, Phase 2 live **31 July**. The interim targets between them
("early June", "mid-July") are working estimates, to be tightened as each phase
starts.

### Guiding principle — phase by ongoing workload

Features differ in how much *ongoing human work* they create once live. The
build is sequenced so each phase is launchable without waiting on a staffing
decision the steering committee hasn't made yet:

- **Phase 1** — downloads, static pages, light-touch forms. No ongoing load.
- **Phase 2** — the data dashboard, refreshed on a monthly cycle that is mostly
  automated and feeds the newsletter.
- **Phase 3** — the organisational-action campaign, which runs in a defined,
  time-limited window rather than as an always-on inbox.

The reframe of the testimony feature (see Phase 3) means no feature in this
plan now requires permanent moderation or a safeguarding rota.

## Phase 2 — Data dashboard with a monthly update cycle

*Rated 4.0, and framed in Round 1 as the manifesto's "live argument" — NHS and
government data set against promises-versus-delivery. The design here is what
keeps it from going stale: a **monthly refresh**, mostly automated via a data
agent and NotebookLM, where each refresh also produces a **factoid for the
BMHWA newsletter**. The newsletter's own cadence is the forcing function — the
dashboard stays current because the newsletter needs the number. **Live by 31 July.***

### ⛔ Gate A — unlocks Phase 2

**BMHWA must agree:**
1. The **data sources** the dashboard draws on (which metrics, from where).
2. Who **runs the monthly cycle** — a light role (~1 hr/month: trigger the
   refresh, sanity-check, write the factoid).

| # | Milestone | Owner | Depends on | Target |
|---|---|---|---|---|
| 2.1 | Dashboard scope + data sources agreed | `[JOINT]` | Gate A | early July |
| 2.2 | Dashboard + monthly update pipeline (data agent + NotebookLM) built on staging | `[BLKOUT]` | 2.1 | mid-July |
| 2.3 | First monthly cycle run jointly as a test — produces a dashboard update and a newsletter factoid | `[JOINT]` | 2.2 | late July |

> Where data sources publish official APIs or downloadable datasets (ONS, NHS
> Digital, etc.), the pipeline should use those rather than scrape — more
> reliable month-on-month. Scraping is reserved for sources with no feed.

## Phase 3 — Organisational-action campaign & signatory directory

*The reframed testimony feature: a **time-limited campaign** gathering accounts
of what organisations have done to progress manifesto commitments — not
personal mental-health stories. BLKOUT and BMHWA solicit a couple of seed

## Scope boundaries

To keep expectations clear on both sides, this plan covers the build only.

**BLKOUT is delivering:** the website build across all phases; the in-house
design using the Windrush typeface; the monthly dashboard update pipeline;
editor documentation; and a defined handover.

**BLKOUT is not delivering:** content writing, translation, or open-ended
maintenance after handover. The monthly dashboard cycle is handed to a named
BMHWA owner at milestone 2.5.

**Resolved across earlier drafts:** the testimony reframe means there is no
longer a safeguarding framework or permanent moderation rota in scope — both
are removed. The mailing-list "Survey B" is no longer a separate parked item;
it is folded into Phase 3 as the campaign's intake channel.
